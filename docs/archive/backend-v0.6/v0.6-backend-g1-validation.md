# v0.6 后端 G1：验收记录

- 验收范围：I1 执行平面无行为重构
- 回滚基线：`ea534bc`（v0.5.6 发布元数据）
- `frontend/`：不属于本后端变更集，禁止进入后端提交

## `/health` smoke 归因

`/health` smoke 在 G1 验收时未通过，真实响应为 HTTP 404。该结果是 **基线既存缺陷**，不是 I1 回归：

1. G1 当时的 `linrouter_core/runtime/http_api_runtime.py` 与 `ea534bc` 完全一致；
2. 两个版本当时均先以静态资源兜底处理非 `/api/`、非 `/v1/` 的路径，后续 `/health` 分支不可达；
3. G1 当时隔离 `create_server` 的真实 HTTP 请求复现 `/health` 为 404。

按无行为重构边界，I1 未调整路由顺序，也未将该 smoke 计为通过。

## 后续独立 P0：基线缺陷修复

`/health` 路由可达性已作为 I1 之外的独立 P0 修复处理：仅将既有 `/health` 判定移动到静态资源兜底之前，不改变 `/`、`/api/*`、`/v1/*` 或静态资源的既有分流。真实 `create_server` HTTP smoke 覆盖 `/health` 返回 `200` 且 `ok: true`，并同时回归上述分流路径。

## G1 历史验收结论

- I1 执行平面范围内的自动化、专项、源码边界和 Codex 复核按已有记录通过；
- G1 当时 `/health` 不作为通过证据，登记为基线 smoke 缺口；
- 因该当时未修复的基线缺口，G1 未宣布全绿通过；后端 I1 变更已独立归档，可回滚至 `ea534bc`。该缺口已由上述独立 P0 修复覆盖。

# v0.6 执行面宽桥依赖与九类端口 owner 映射

- 盘点日期：2026-07-12
- 审计范围：当前 `main` 工作树的生产 Python 源码；未包含未跟踪 `frontend/`。
- 基线：`9218abc`（文档冻结）/ `38689bb`（I3 代码）；本清单只盘点和规划，不改变运行语义。
- 宽桥定义：`linrouter_core/contracts/execution_ports.py:74-140` 的 `ExecutionDependencies`。它将 50+ 个跨域能力以 `self` 注入 `CandidateRuntime`，是本次唯一待删除的执行面宽桥。

## 4. 可执行集中迁移清单（单一最终集成提交）

1. 在 contracts 层新增 #3/#6/#8/#9 的端口与 frozen DTO；将 #5 补强为 lease 语义。不要新增万能 `router: Any`、`RuntimePort` 或 dict 分类结果。
2. 新增 `RequestPreparationService`、stream reader/usage adapter，并让它们只依赖现有 `UpstreamAdapter`/稳定 DTO；为 `WafLockState` 实现 lease 端口适配。
3. 为 `CandidateHealthService` 补齐 runtime 实际需要的 query/health 窄方法；保持它对 `ConfigStore` 的唯一状态所有权。保留 `ExecutionPolicyService` 的当前唯一策略所有权。
4. 为 `ObservabilityService` 与 `DebugCapture` 建立直接端口适配，保留当前字段、脱敏和 best-effort 行为。
5. 重写 `CandidateRuntime.__init__`，显式注入九类端口/DTO；先等价替换非流与流的每一处 `router.*` 访问，再删除 `self.dependencies` 与 `router = ...`。
6. 令 `NonStreamExecutionService`、`StreamExecutionService` 仅保存 runtime；令 `ArkProxyRouter` 只负责组合现有 owner/新 adapter 和历史 `call/stream` 薄委托。
7. 删除 `ExecutionDependencies` 及 `CandidateErrorClassifier` 宽桥兼容路径；静态断言生产源码不再出现该名字，也不反向 import `app`、`RouterHandler` 或 HTTP transport。
8. 在同一提交中运行完整回归和真实 server smoke；不得混入 `frontend/`、配置、日志或构建产物。

## 6. 风险与决定

- #1 在冻结文档中将 `CandidateQueryPort + HealthStatePort` 视为一个 owner 域；本清单按“九类”计为一个领域，不虚构第十个业务 owner。
- `ExecutionDependencies` 当前不只是类型桥：runtime 直接使用其方法，因此只能在九类实现全部接线后一次删除，不能保留双路径。
- `CandidateRuntime` 的候选循环、fallback 判断、HTTP/stream iterator 生命周期和日志语义属于冻结行为；迁移只允许将依赖读取换成端口调用。
- `ArkProxyRouter` 中仍有手动探测、HTTP handler/API 支持等非执行调用；本批只移走上述 execution runtime 消费，避免把无关 API/handler 逻辑带入最终收敛。
